## 対象API
- 検索契約（クライアント実行, [[DD-APP-API-006]]）

## 対象UC
- [[RQ-UC-003]]（利用者がキーワードで動画を探す）
- [[RQ-UC-004]]（利用者がタグで動画を絞り込む）
- [[RQ-UC-005]]（利用者が日付・再生時間で絞り込む）

## 生成元PWモデル
- [[IT-PW-UC-003]]
- [[IT-PW-UC-004]]
- [[IT-PW-UC-005]]

## テスト目的
- 検索・タグ・範囲条件の複合評価契約を検証する。

## 契約化した受入条件
- キーワードAND条件がタイトル/概要/タグへ一貫適用される。
- タグ条件（AND/OR）で件数変化が説明可能である。
- 日付・再生時間の境界不正は適用拒否される。

## 因子（機能要件ベース）
| 因子 | 関連要求 | 水準 |
| --- | --- | --- |
| キーワード条件 | [[RQ-FR-010]] | 空, 単語1件, 複数語AND |
| タグ条件 | [[RQ-FR-008]] | なし, OR, AND |
| 日付範囲 | [[RQ-FR-011]] | なし, 正常, 境界不正 |
| 再生時間範囲 | [[RQ-FR-012]] | なし, 正常, 境界不正 |

## テストケース一覧
| case_id | 条件 | 期待結果 |
| --- | --- | --- |
| IT-CASE-005-C01 | キーワード単独 | 一致件数が変化 |
| IT-CASE-005-C02 | タグOR + キーワード | 緩い絞り込み結果を返す |
| IT-CASE-005-C03 | タグAND + 範囲条件 | 厳しい絞り込み結果を返す |
| IT-CASE-005-C04 | 日付境界不正 | 条件未適用 + 入力エラー表示 |
| IT-CASE-005-C05 | 再生時間境界不正 | 条件未適用 + 入力エラー表示 |

## 手順
1. 一覧表示後、キーワード/タグ/日付/再生時間を順に適用する。
2. 並び順を切り替えて先頭データを確認する。
3. 条件を解除して全件表示へ戻す。

## 期待結果
- 条件適用ごとに結果件数が変化する。
- 並び順切替が確実に反映される。
- クリア操作で初期状態へ復帰する。
