## 対象API
- `POST /api/v1/ops/ingestion/runs/{runId}/retry`（[[DD-APP-API-008]]）

## 対象UC
- [[RQ-UC-008]]（管理者が[[RQ-GL-011|再収集]]を実施する）

## 生成元PWモデル
- [[IT-PW-UC-008]]

## テスト目的
- [[RQ-GL-011|再収集]]契約（対象検証、再実行起動、上限/競合制御）を検証する。

## 契約化した受入条件
- 対象runが有効な場合は `retryRunId` を返却する。
- 同一対象で再実行中の場合は重複起動を拒否する。
- 上限超過や対象不正は起動拒否し、理由を返却する。

## 因子（機能要件ベース）
| 因子 | 関連要求 | 水準 |
| --- | --- | --- |
| 親run状態 | [[RQ-FR-016]] | `failed`, `partial`, 不正状態 |
| 再実行競合 | [[RQ-FR-016]] | なし, あり |
| 再実行上限 | [[RQ-FR-016]] | 上限内, 上限超過 |

## テストケース一覧
| case_id | 条件 | 期待結果 |
| --- | --- | --- |
| IT-CASE-007-C01 | `failed` run + 上限内 | `202` + `retryRunId` |
| IT-CASE-007-C02 | 対象不正 | 起動拒否 + 理由返却 |
| IT-CASE-007-C03 | 再実行競合 | 重複起動拒否 |
| IT-CASE-007-C04 | 上限超過 | 起動拒否 |

## 手順
1. 失敗runを作成する。
2. [[RQ-GL-011|再収集]]APIを実行して `retryRunId` を取得する。
3. 状態APIで[[RQ-GL-011|再収集]]run完了を確認する。

## 期待結果
- [[RQ-GL-011|再収集]]runが起動し、追跡可能である。
- 上限超過時に拒否応答が返る。
