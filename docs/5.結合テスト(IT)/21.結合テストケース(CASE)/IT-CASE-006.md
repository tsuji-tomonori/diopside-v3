## 対象API
- 動画詳細組み立て契約（[[DD-APP-API-007]]）

## 対象UC
- [[RQ-UC-006]]（利用者が動画詳細を確認する）

## 生成元PWモデル
- [[IT-PW-UC-006]]

## テスト目的
- 詳細表示契約（詳細データ、[[RQ-GL-016|コメント密度波形]]、[[RQ-GL-017|ワードクラウド]]、外部遷移）の整合を検証する。

## 契約化した受入条件
- 選択動画IDに対して詳細情報が一貫して返却される。
- [[RQ-GL-016|コメント密度波形]]未生成時は代替表示で継続利用できる。
- [[RQ-GL-017|ワードクラウド]]未生成時もモーダル利用を継続できる。

## 因子（機能要件ベース）
| 因子 | 関連要求 | 水準 |
| --- | --- | --- |
| 詳細情報充足 | [[RQ-FR-013]] | 完全, 一部欠落 |
| 波形データ状態 | [[RQ-FR-020]] | 生成済み, 未生成 |
| [[RQ-GL-017|ワードクラウド]]状態 | [[RQ-FR-021]] | 生成済み, 未生成 |
| 外部遷移可否 | [[RQ-FR-014]] | 正常URL, 異常URL |

## テストケース一覧
| case_id | 条件 | 期待結果 |
| --- | --- | --- |
| IT-CASE-006-C01 | 詳細/波形/[[RQ-GL-017|ワードクラウド]]正常 | モーダル表示と遷移成功 |
| IT-CASE-006-C02 | 波形未生成 | 代替表示 + モーダル継続 |
| IT-CASE-006-C03 | [[RQ-GL-017|ワードクラウド]]未生成 | 代替表示 + モーダル継続 |
| IT-CASE-006-C04 | 外部遷移異常 | 失敗通知 + URLコピー導線 |

## 手順
1. 一覧の動画カードを選択しモーダル表示を確認する。
2. タグ解決と公開日時表示を確認する。
3. `YouTubeで開く` 操作で遷移URLを確認する。

## 期待結果
- 選択動画IDとモーダル情報が一致する。
- 遷移URLのvideoIdが一致する。
