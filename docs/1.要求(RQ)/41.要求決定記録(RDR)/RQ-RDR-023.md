## 決定事項
- NFRと制約の整合確認で抽出した要確認事項について、要求本文に判定条件と適用範囲を追記する。
- [[RQ-AV-001]] / [[RQ-PS-001]] / [[RQ-OBY-001]] / [[RQ-INT-001]] / [[RQ-DATA-001]] は、運用体制・コスト上限・APIクォータ制約と両立する例外条件を明記する。
- [[RQ-UX-003]] / [[RQ-UX-014]] / [[RQ-UX-016]] / [[RQ-UX-020]] / [[RQ-UX-022]] / [[RQ-UX-023]] は、MVP適用条件と[[RQ-SH-001|管理者]]運用前提を明示する。

## 理由
- 要件自体は制約と矛盾していないが、運用条件が未定義のままでは[[AT-GO-001|リリース判定]]の再現性が低い。
- 個人開発体制（[[RQ-PC-004]]）とコスト上限（[[RQ-PC-006]]）を前提に、Must要件を維持しつつ実行可能性を確保する必要がある。
- スコープ境界（[[RQ-SC-001]]）に未定義の機能を含むUX要件は、適用条件を明示しないと過剰要求となる。

## 影響
- 要求文書: 上記11件のNFRを意味変更としてPATCH更新する。
- 受入運用: [[AT-PLAN-001]] / [[AT-RPT-001]] で、運用時間帯・例外条件・対象導線の判定を記録可能にする。
- 制約整合: [[RQ-PC-002]] / [[RQ-PC-004]] / [[RQ-PC-006]] とNFRのトレースを要求文書上で明示する。
