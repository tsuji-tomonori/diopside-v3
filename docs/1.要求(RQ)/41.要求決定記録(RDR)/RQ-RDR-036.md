## 決定事項
- 配信結果に基づくタグ付けは自動化せず、[[RQ-SH-001|管理者]]が管理画面で実施する方針を維持する。
- 管理画面は、収集結果からタグ付けに必要な情報をコピペ可能な形式で提示し、外部LLM（ChatGPT）へ貼り付けて提案生成できる運用を標準化する。
- 外部LLMの出力はJSON契約で管理画面へアップロードし、検証成功時のみDB正本へ反映する。
- DB反映後は [[RQ-GL-008|タグマスター]] と [[RQ-GL-009|アーカイブ索引]] の再生成・公開切替を同一運用チェーンで実行する。

## 理由
- 配信結果の意味解釈には運用判断が必要であり、完全自動付与は誤分類時の影響が大きい。
- [[RQ-SH-001|管理者]]作業では、収集結果の再整形と転記コストが高いため、コピペ可能な標準入力を管理画面で提供する必要がある。
- 外部LLM出力をそのまま反映すると品質事故につながるため、JSON契約検証を反映前の必須ゲートにする必要がある。

## 影響
- 機能要求: [[RQ-FR-019]] に、LLM支援タグ運用（入力生成・JSON取込・検証・反映）を追加する。
- ユースケース: [[RQ-UC-009]] を「収集結果確認 -> コピペ -> JSON取込 -> DB反映 -> 配信更新」フローへ拡張する。
- タグ運用: [[RQ-FR-005]] と整合し、DB正本更新後の配信再生成対象を [[RQ-GL-008|タグマスター]] のみでなく [[RQ-GL-009|アーカイブ索引]] まで含める。
- 設計: [[BD-APP-API-002]] / [[DD-APP-API-013]] で入出力契約、バリデーション、監査記録を具体化する。
