## 決定事項
- 公開UI（検索/一覧/詳細）は、端末優先順位を「スマートフォン -> タブレット -> PC」で固定する。
- [[RQ-GL-012|受入判定]]は、優先順位に沿って390px、768px、1280pxの順で主要操作成立を確認する。
- 管理画面は本決定の適用対象外とし、運用UIの端末最適化方針は別決定で扱う。

## 理由
- 利用者導線の中心は公開UIであり、モバイル利用時の操作欠損を先に排除することが検索到達性の改善に直結する。
- 端末優先順位を要求段階で固定することで、BD/DD/ATの判定順序を統一できる。
- タブレット/PCの成立条件を後追いではなく同時に定義することで、モバイル最適化による回帰を抑制できる。

## 影響
- 要求文書: [[RQ-UX-001]] と [[RQ-UX-005]] の受入基準を端末優先順で更新する。
- 設計文書: [[BD-APP-UI-002]] と [[DD-APP-UI-002]] で、モバイル先行のレイアウト・状態遷移・操作制約を明文化する。
- 受入運用: [[AT-SCN-002]] で端末別の確認順序と判定記録を維持できるようにする。
