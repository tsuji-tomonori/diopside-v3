## 決定事項
- 配信前（upcoming状態）の動画メタデータ取得と、配信後の同一動画に対する再取得・差分確認を自動実行対象とする。
- 配信結果に基づくタグ付けは自動化せず、管理者の後続運用で実施する。
- 自動処理の完了状態は差分あり・差分なしを区別して記録し、再確認失敗時は再試行可能な状態で保持する。

## 理由
- 配信前後でタイトル・説明欄・開始時刻が更新されるケースがあり、単回収集では差分取りこぼしが発生する。
- 過去検証実装（`.workspace/NotifyDeliveryScheduleApp`）で、配信前後の状態差分を識別する実装方針が確認できる。
- 配信結果タグは運用判断を含み自動判定誤りの影響が大きいため、管理者手動で品質を担保する。

## 影響
- 要求文書: `[[RQ-FR-019]]` を新規追加し、配信前後の自動再確認と手動タグ運用の境界を明文化する。
- ユースケース: `[[RQ-UC-009]]` を追加し、配信前確認→配信後再確認→手動タグ付けの運用シナリオを要求と結合する。
- 既存要求: `[[RQ-FR-004]]`（正規化）、`[[RQ-FR-005]]`（タグ辞書）、`[[RQ-FR-017]]`（実行結果確認）との責務分担を明確化する。
- 運用: 再確認結果は run 履歴で追跡し、タグ付け作業は管理者の運用フローとして記録する。
