## 決定事項
- [[RQ-AV-001]] 〜 [[RQ-DATA-001]] の10件を、共通テンプレート文からカテゴリ固有の要件へ更新する。
- 各NFRは SnowCard の `受入基準` に数値閾値を持ち、ATで計測可能な判定条件を明示する。
- 各NFRの `related` は実際の設計/運用文書へ接続し、一律参照を解消する。

## 理由
- 現行NFRは文言重複が多く、要件間の差分や[[RQ-GL-012|受入判定]]の観点が読み取れなかった。
- 収集運用と検索品質の判定を再現可能にするには、カテゴリ別の測定指標が必要である。
- 要求段階で計測基準を固定することで、BD/DD/ATの追従更新を容易にする。

## 影響
- 要求文書: [[RQ-AV-001]] / [[RQ-PS-001]] / [[RQ-SEC-001]] / [[RQ-PRC-001]] / [[RQ-OBY-001]] / [[RQ-DEV-001]] / [[RQ-INT-001]] / [[RQ-COST-001]] / [[RQ-UX-001]] / [[RQ-DATA-001]] を意味変更としてPATCH更新する。
- 受入運用: [[AT-PLAN-001]] / [[AT-RPT-001]] / [[AT-GO-001]] でNFR判定の記録粒度を維持する。
- 設計文書: [[BD-SYS-QUAL-001]] / [[BD-SYS-SEC-001]] / [[BD-DEV-PIPE-001]] / [[BD-APP-DATA-001]] などの関連設計で閾値トレース可能性を確保する。
