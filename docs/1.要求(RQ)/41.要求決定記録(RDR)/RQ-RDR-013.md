## 決定事項
- `[[RQ-DM-001]]` 〜 `[[RQ-DM-007]]` を雛形記述から具体定義へ更新し、属性・関連・整合条件を明示する。
- 各モデルで主キー、外部キー、論理識別子を定義し、`[[RQ-GL-002|収集ジョブ]]` 実行履歴に追跡可能な構造を必須とする。

## 理由
- 既存のドメインモデル記述では、要求・設計・テスト間でデータ境界を一意に解釈できなかった。
- `DD-API` 群で定義済みの run / retry / tag 契約と整合する要求側モデルが必要になった。

## 影響
- 要求文書: `[[RQ-DM-001]]` 〜 `[[RQ-DM-007]]` の意味を具体化し、`version` をPATCH更新する。
- 設計文書: `[[BD-DATA-001]]`, `[[BD-ERD-001]]` との対応確認を実施する。
- テスト文書: run起点の追跡観点（UT/IT/AT）を参照可能にする。
