## SnowCard（日本語）
- 要求ID: RQ-PS-001
- 種別: 非機能要求
- 優先度: MUST
- 要求: 本システムは、検索と一覧操作を体感待ちの少ない性能で提供すること。
- 根拠: 公開アーカイブ探索では操作遅延が離脱に直結し、[[RQ-GL-014|検索条件]]の試行回数が低下するため。
- 受入基準:
  - 主要ページ初回表示時間（LCP相当）が p95 で 2.5秒以下である。
  - キーワード検索またはタグ絞り込み適用後、結果再描画完了までが p95 で 1.5秒以下である。
  - [[RQ-GL-010|段階ロード]] の [[RQ-GL-009|archive_index]] 読み込み開始から結果表示までが p95 で 2.0秒以下である。
  - 50件表示時の並び替え操作は p95 で 1.0秒以下である。
  - 計測条件は4G相当（下り10Mbps/上り3Mbps/RTT 50ms）、ミドルレンジ端末、同時利用10未満を標準とする。
- 例外/エラー:
  - 性能閾値を連続3回超過した場合は劣化インシデントとして記録し、次リリースまでに改善策を実装する。
  - 外部ネットワーク要因で計測不能なケースは除外理由を [[AT-RPT-001]] に記録する。
  - [[RQ-PC-006]] の上限超過見込み時は、p95閾値を維持したまま [[RQ-GL-010|段階ロード]] の粒度とキャッシュ期間を見直し、常時稼働リソースを追加しない。
