## SnowCard（日本語）
- 要求ID: RQ-DEV-002
- 種別: 非機能要求
- 優先度: MUST
- 要求: 本システムは、AIエージェント運用において役割分離・最小権限・実行上限を標準化し、再現可能な設計判断と安全な実行を維持すること。
- 根拠: [[RQ-PC-007]] と [[RQ-PC-009]] のAI支援開発前提および低リスクリリース制約を満たすため。
- 受入基準:
  - 変更計画と実装を分離し、Primary運用は「分析・方針確定」と「実装・検証」で役割を分けて実施される。
  - レビュー専用/監査専用のエージェントは `edit` を禁止し、差分確認に必要な読み取り操作のみ許可される。
  - 危険度の高い操作（例: 外部公開、破壊的削除、権限昇格）はデフォルトで `ask` または `deny` とし、常時許可しない。
  - サブエージェント呼び出しは許可対象を明示し、未定義の呼び出しを許可しない。
  - 調査系エージェントは実行ステップ上限を持ち、上限到達時は未完了タスクと次アクションを記録する。
  - 要求追加または意味変更時は [[RQ-RDR-029]]、設計追加または意味変更時は [[BD-ADR-016]] を同一変更で更新する。
- 例外/エラー:
  - 権限定義が未設定または過剰許可のエージェントは、本番運用で使用しない。
  - 実行ログまたは判断根拠を残せない変更は採用しない。
