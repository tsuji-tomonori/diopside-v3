## SnowCard（日本語）
- 要求ID: RQ-DEV-004
- 種別: 非機能要求
- 優先度: SHOULD
- 要求: 本システムは、AWSサービス追加・データモデル変更・契約変更時にアーキテクチャ決定記録（ADR）を作成し、既存BD-SYS-ADR体系と統合管理すること。
- 根拠: 個人開発では意思決定の経緯が記憶に依存しやすく、ADRで形式知化することで将来の判断根拠を保全するため。
- 受入基準:
  - 以下の変更時にADRが作成されている: 新規AWSサービス追加、データモデルの構造変更、配信JSON契約の破壊的変更。
  - ADRに背景・選択肢・決定・結果の4セクションを含む。
  - ADRは既存の `BD-SYS-ADR` 体系の命名規則と配置に従う。
  - ADR未作成の該当変更がリリース候補に含まれる場合、レビューチェックリストで検知する。
- 例外/エラー:
  - 緊急障害対応で事前ADR作成が困難な場合は、復旧後7日以内に事後ADRを作成する。
  - 軽微な設定変更（閾値調整、タグ値変更等）はADR対象外とし、変更履歴への記録で代替する。
