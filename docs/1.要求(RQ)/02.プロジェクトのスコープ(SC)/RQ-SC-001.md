## 収集対象判定ルール
- 「公開」状態の動画のみを対象とする（非公開・限定公開は対象外）。
- 区分は次で固定する。
  - [[RQ-GL-003|公式投稿動画]]: 投稿チャンネルが `白雪 巴/Shirayuki Tomoe`。
  - [[RQ-GL-004|出演動画]]: 投稿チャンネルが公式以外で、出演タグまたは運用確認で出演が確定。
- Shorts/通常動画/配信アーカイブは、公開であれば同一ルールで対象に含める。
- 削除済み動画・視聴不可動画は保持データ上でステータス管理し、再取得時に整合させる。

## 非対象（Out of Scope）
- 非公開/限定公開動画の取得。
- 著作権保護対象データの再配布。
- YouTube動画ファイル本体の保存。
- コメント本文やチャットログ等の生データの恒久保存・再配布。
- 推薦アルゴリズムやパーソナライズ機能。
- 課金機能、会員機能、外部へのデータ販売。

## スコープ境界（MVP）
- MVPで保証するのは「収集対象定義に沿ったデータ反映」と「検索で目的動画に到達できること」。
- MVP最小保証範囲は、エピックでは [[RQ-SC-002]] / [[RQ-SC-004]] / [[RQ-SC-005]] とする。
- 高度検索（自然言語ランキング、意味検索）はMVP外とし、将来拡張に回す。
- 収集処理の内部実装（ジョブ実行基盤の具体製品選定）は設計文書で決定する。

## ステークホルダー
- 個人開発体制のため、ステークホルダーは [[RQ-SH-001]]（[[RQ-SH-001|管理者]]）と [[RQ-SH-002]]（[[RQ-SH-002|利用者]]）の2者に限定する。
- 設計・運用・品質保証・セキュリティの責務は[[RQ-SH-001|管理者]]に統合し、[[RQ-SH-002|利用者]]は利用評価とフィードバックを担う。

## 完了条件（Definition of Done）
- Must機能要求（[[RQ-FR-001]] 〜 [[RQ-FR-025]]）が受入シナリオで再現可能である。
- 非機能要求（可用性/性能/セキュリティ/可観測性/コスト）の最低基準がATで確認できる。
- 収集対象判定ルールの運用手順が [[AT-PLAN-001]] と [[AT-RUN-001]] に反映されている。
- 要求追加・意味変更時にRDRが同一変更で更新されている。

## 変更運用（エピック単位）
- 要求追加・スコープ変更は、まず影響するEpic（[[RQ-SC-002]]〜[[RQ-SC-008]]）を割り当てる。
- 次に影響する境界（Ingestion / TagManagement / Publishing / Viewing / Administration / Analytics）と、Published Language変更有無を判定する。
- [[RQ-GL-007|bootstrap]] / [[RQ-GL-008|タグマスター]] / [[RQ-GL-009|アーカイブ索引]] の契約変更は、RDR/ADR/検証文書を同一変更で更新する。
- FRのエピック割当変更時は [[RQ-RTM-003]] と本書を同一変更で更新する。
