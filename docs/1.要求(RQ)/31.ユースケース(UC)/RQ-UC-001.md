## 概要
- [[RQ-SH-001|管理者]]が管理画面で収集対象と実行条件を指定して[[RQ-GL-002|収集ジョブ]]を起動し、実行受付から完了確認までを行う利用シナリオ。

## 基本フロー
1. [[RQ-SH-001|管理者]]が管理画面の収集実行画面を開き、取得モード（公式取り込み/出演補完取り込み/差分更新）、対象区分（[[RQ-GL-003|公式投稿動画]]/[[RQ-GL-004|出演動画]]）、実行範囲を指定する。
2. [[RQ-SH-001|管理者]]が実行を確定し、システムは入力値と実行可否（同時実行制約）を検証する。
3. システムは実行を受け付け、run ID を払い出して状態を `queued` から `running` へ遷移させる。
4. [[RQ-SH-001|管理者]]は進行状況（処理件数/成功件数/失敗件数）とイベントログを確認する。
5. システムは完了時に最終状態を `succeeded` または `failed` として確定し、[[RQ-SH-001|管理者]]が結果を記録する。

## 代替/例外フロー
- 入力不備: 対象区分や範囲指定が不正な場合、実行を受け付けず必須項目エラーを表示する。
- 実行競合: 既存の実行中ジョブがある場合、二重起動を拒否して既存runの監視画面へ誘導する。
- 外部要因失敗: API制限・通信失敗等で失敗した場合、失敗理由と[[RQ-UC-008|再収集実施]]導線を表示する。
