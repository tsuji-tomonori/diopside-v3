## SnowCard（日本語）
- 要求ID: RQ-FR-018
- 種別: 機能要求
- 優先度: MUST
- 要求: 本システムは、管理者が失敗または未取得対象に対して手動で[[RQ-GL-011|再収集]]を実行できることを提供する。
- 根拠: 外部要因で失敗した収集を運用判断で回復し、欠落データを最小化するため。
- 受入基準:
  - 管理者は run または動画単位で[[RQ-GL-011|再収集]]対象を選択して実行できる。
  - [[RQ-GL-011|再収集]]結果が元runと関連付けられ、改善状況を比較できる。
- 例外/エラー:
  - 既に[[RQ-GL-011|再収集]]中の対象を重複指定した場合は受け付けない。
  - [[RQ-GL-011|再収集]]上限回数を超える対象は手動確認キューへ送る。
- 依存・関連:
  - [[RQ-UC-008]]
  - [[RQ-RDR-006]]
  - [[BD-API-001]]

## テスト観点
- run単位・動画単位の[[RQ-GL-011|再収集]]が正しく実行されること。
- 重複指定と上限超過で想定どおりに拒否/キュー移送されること。

## 個別確認チェックリスト
- 確認主体: 管理者
- 重点確認対象: [[RQ-GL-011|再収集]]対象選択と重複防止
- 確認方法: 重複指定拒否と上限超過時キュー移送を確認する。
