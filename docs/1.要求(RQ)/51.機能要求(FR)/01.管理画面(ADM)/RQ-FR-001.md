## SnowCard（日本語）
- 要求ID: RQ-FR-001
- 種別: 機能要求
- 優先度: MUST
- 要求: 本システムは、[[RQ-SH-001|管理者]]が管理画面から公開動画の[[RQ-GL-002|収集ジョブ]]を「公式取り込み」「出演補完取り込み」「差分更新」の取得モードで起動し、実行状態と処理結果を確認できることを提供する。
- 根拠: 公開アーカイブの継続運用では、定期実行だけでなく任意タイミングの収集起動が必要であるため。
- 受入基準:
  - 実行受付時に run ID が払い出され、状態が `queued` から `running` へ遷移する。
  - [[RQ-SH-001|管理者]]は実行モードごとの処理件数（対象件数・追加件数・補完件数・失敗件数）を実行中および完了後に確認できる。
  - 実行結果には対象動画の更新種別（新規/既存/補完/再確認）が記録される。
- 例外/エラー:
  - 同時実行制約に抵触した場合は新規実行を受け付けず、既存 run を案内する。
  - 未定義モードまたはモード必須入力不足の場合は実行を受け付けず入力エラーとして返す。
  - 対象区分・実行範囲が不正な場合は実行せず入力エラーとして返す。
- 依存・関連:
  - [[RQ-UC-001]]
  - [[RQ-RDR-006]]
  - [[BD-API-001]]

## テスト観点
- 収集対象区分・期間指定の境界値で実行可否が正しく判定されること。
- 取得モードごとに必要入力が検証され、結果集計項目が欠落しないこと。
- 実行競合時に二重起動せず既存 run 参照導線へ遷移すること。

## 個別確認チェックリスト
- 確認主体: [[RQ-SH-001|管理者]]
- 重点確認対象: run ID と実行状態遷移
- 確認方法: 実行受付ログと進行状況表示を照合する。
