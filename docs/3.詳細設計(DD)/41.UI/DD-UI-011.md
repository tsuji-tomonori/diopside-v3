## 詳細仕様
- 本画面は [[DD-UI-011|UI-A05]] として、公開後の到達性確認と運用記録を扱う。
- 配信経路異常時は、監視runへ戻る再確認導線を提供する。

## 画面構成
| ブロック | 内容 | 操作 |
| --- | --- | --- |
| 公開状態 | 最終公開時刻、公開run結果 | 状態確認 |
| 経路チェック | `/web/`, `/docs/`, `/openapi/`, `/api/` 到達性 | 再確認実行 |
| 異常記録 | 影響範囲、暫定対応、担当者 | 記録保存 |

## 状態遷移
- `checking`: 経路確認実行中。
- `healthy`: 全経路到達可。
- `degraded`: 一部経路異常。
- `failed`: 全体異常または確認失敗。

## 操作制約
- 経路確認中は再実行ボタンを無効化する。
- `degraded/failed` 時は [[DD-UI-006|UI-A02]] への戻り導線を表示する。
- 重大異常記録は担当者未選択のまま保存できない。

## I/Oまたは責務
- 入力: 経路確認結果、公開run結果、管理者記録入力。
- 出力: 到達性判定、運用記録、監視画面遷移イベント。
