## 詳細仕様
- 運用ステータス画面は収集実行状況、失敗履歴、公開反映結果を統合表示する管理UIとする。
- 破壊的操作は二段階確認（確認ダイアログ + 実行後トースト）を必須にする。

## 表示ブロック
| ブロック | 主要表示 | 操作 |
| --- | --- | --- |
| 実行サマリ | 最新run状態、処理件数、失敗件数 | 詳細ドリルダウン |
| 実行履歴 | run一覧、開始/終了時刻、失敗理由 | 再実行 |
| 公開反映 | 最終公開時刻、invalidation状態 | 再確認 |
| 障害通知 | 重大エラー、未解決件数 | 失敗ログ参照 |

## 操作ルール
- 収集起動は同種run実行中は無効化し、`409` 理由を表示する。
- 再実行は失敗runのみ許可し、成功runには操作を出さない。
- 公開確認は `docs:deploy` 実行後にのみ有効化する。

## 異常時表示
- API失敗時はブロック単位でエラー表示し、他ブロック操作を継続可能とする。
- 認証切れ時は画面全体を読み取り専用に切り替え、再認証導線を表示する。
- 連続失敗3回で障害アラートバナーを固定表示する。

## I/Oまたは責務
- 入力: run状態API、再実行API、公開確認API、運用操作イベント。
- 出力: 管理画面表示状態、実行/再実行要求、障害通知表示、監査ログ連携。
