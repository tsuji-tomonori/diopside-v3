## 詳細仕様
- 運用ステータス画面（[[DD-UI-006|UI-A02]]）は、run進行監視と失敗履歴確認のハブ画面とする。
- 実行起点は [[DD-UI-007|UI-A01]]、[[RQ-GL-011|再収集]]設定は [[DD-UI-008|UI-A03]] へ遷移する。
- 画面部品は [[DD-UI-017|RunStatusScreen]] と [[DD-UI-018]] を利用する。

## 表示ブロック
| ブロック | 主要表示 | 操作 |
| --- | --- | --- |
| 実行サマリ | 最新run状態、処理件数、失敗件数 | 詳細ドリルダウン |
| 実行履歴 | run一覧、開始/終了時刻、失敗理由 | run選択 |
| 失敗明細 | 失敗動画、失敗分類、再試行可否 | [[DD-UI-008|UI-A03]] へ遷移 |
| 障害通知 | 重大エラー、未解決件数 | ログ参照/再読込 |

## 操作ルール
- `failed` runのみ [[DD-UI-008|UI-A03]] への遷移を有効化する。
- `running` runは30秒間隔で自動再読込し、手動再読込も許可する。
- 同一runを重複選択した場合は再描画のみで副作用を発生させない。

## 異常時表示
- API失敗時はブロック単位でエラー表示し、他ブロック操作を継続可能とする。
- 認証切れ時は画面全体を読み取り専用に切り替え、再認証導線を表示する。
- 連続失敗3回で障害アラートバナーを固定表示する。
- 監視API遅延時は直近成功スナップショットを保持し、取得時刻を明示する。

## I/Oまたは責務
- 入力: run状態API、結果明細API、運用操作イベント。
- 出力: run監視状態、run選択イベント、[[RQ-GL-011|再収集]]画面遷移、障害通知表示。
