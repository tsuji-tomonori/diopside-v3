## 詳細仕様
- 運用ステータス画面は収集実行状況、失敗履歴、公開反映結果を統合表示する管理UIとする。
- 破壊的操作は二段階確認（確認ダイアログ + 実行後トースト）を必須にする。
- 配信前後確認導線では、外部LLM向け提案入力のコピーと結果JSON取込を同一画面コンテキストで実行可能にする。

## 表示ブロック
| ブロック | 主要表示 | 操作 |
| --- | --- | --- |
| 実行サマリ | 最新run状態、処理件数、失敗件数 | 詳細ドリルダウン |
| 実行履歴 | run一覧、開始/終了時刻、失敗理由 | 再実行 |
| LLM提案入力 | 対象動画、差分要約、既存タグ | クリップボードコピー |
| LLM結果取込 | JSON検証結果、反映件数、エラー明細 | JSONアップロード、反映確定 |
| 公開反映 | 最終公開時刻、invalidation状態 | 再確認 |
| 障害通知 | 重大エラー、未解決件数 | 失敗ログ参照 |

## 操作ルール
- 収集起動は同種run実行中は無効化し、`409` 理由を表示する。
- 再実行は失敗runのみ許可し、成功runには操作を出さない。
- 公開確認は `docs:deploy` 実行後にのみ有効化する。
- 提案入力コピーは `video_count > 0` の場合のみ有効化する。
- JSON取込は `schemaVersion` と必須キー検証が成功するまで反映確定を有効化しない。
- JSON反映成功後は公開反映ステップを必須導線として表示し、`tag_master` と `archive_index` の同時更新を促す。

## 異常時表示
- API失敗時はブロック単位でエラー表示し、他ブロック操作を継続可能とする。
- 認証切れ時は画面全体を読み取り専用に切り替え、再認証導線を表示する。
- 連続失敗3回で障害アラートバナーを固定表示する。
- JSON検証失敗時はエラー行番号と理由を一覧表示し、再アップロードまで反映操作を禁止する。

## I/Oまたは責務
- 入力: run状態API、再実行API、タグ提案入力API、タグJSON取込API、公開確認API、運用操作イベント。
- 出力: 管理画面表示状態、実行/再実行要求、コピー用入力、JSON検証結果、障害通知表示、監査ログ連携。
