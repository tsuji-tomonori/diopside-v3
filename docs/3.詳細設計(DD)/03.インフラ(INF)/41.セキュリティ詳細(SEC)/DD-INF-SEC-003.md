## 詳細仕様
- 認証必須経路は `/openapi/*` と `/api/v1/*` に限定し、匿名経路から管理機能へ遷移させない。
- IAM権限は配備実行ロール、運用参照ロール、監査参照ロールに分離する。
- 秘密情報はシークレットストア参照のみ許可し、成果物とログへの出力を禁止する。

## 経路統制
| 経路 | 認証 | 許可操作 |
|---|---|---|
| `/web/*` | 不要 | 閲覧のみ |
| `/docs/*` | 不要 | 閲覧のみ |
| `/openapi/*` | 必須 | 契約確認 |
| `/api/v1/*` | 必須 | 管理操作/参照API |

## 権限統制
- 配備実行ロールは `deploy` と `invalidation` に必要な最小権限のみ許可する。
- 運用参照ロールはログ閲覧・メトリクス閲覧のみ許可し、更新権限を持たせない。
- 監査参照ロールは監査証跡閲覧専用とし、運用変更を禁止する。

## 秘密情報統制
- シークレット値の平文保持を禁止し、環境変数は識別子のみを保持する。
- 例外ログは `error.code` と `trace_id` のみ記録し、秘密値を含む詳細を落とさない。

## 障害時ハンドリング
- 認証失敗率が急増した場合は認証基盤障害として切り分ける。
- 権限不足による配備失敗はロール設定を是正後に再配備する。
