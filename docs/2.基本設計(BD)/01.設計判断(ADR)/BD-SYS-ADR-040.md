## 決定事項
- `docs/2.基本設計(BD)` の章構成を再編し、`01.設計判断(ADR)` をトップレベルへ昇格する。
- `02.全体構成(SYS)` の直下は `11.アーキテクチャ概要(ARCH)` のみとし、ドメイン境界文書はARCH配下へ移設する。
- SYS品質特性は廃止し、品質特性の正本を `[[BD-APP-QUAL-001]]` と `[[BD-INF-QUAL-001]]` に分割する。
- SYSセキュリティ文書は削除し、セキュリティ正本は `[[BD-INF-SEC-001]]` に統一する。

## 理由
- 全体構成配下に複数種別が混在しており、章責務の境界が曖昧だった。
- 品質特性はアプリとインフラで設計・受入観点が異なるため、正本分割の方が追跡性と保守性が高い。
- セキュリティはINF正本に統合済みであり、重複ID維持は参照ノイズを生む。

## 影響
- BD章番号を `01:設計判断 / 02:全体構成 / 03:アプリ / 04:インフラ / 05:開発` へ再配置する。
- `[[BD-APP-QUAL-001]] / [[BD-INF-QUAL-001]]` と `[[BD-INF-SEC-001]]` 参照をRTM/RDR/DD/ADRで置換する。
- 影響確認と整合検証を `reports/impact_check_2026-02-23.md` と `task docs:guard` で記録する。

## 却下した選択肢
- SYS品質特性を廃止文書として残す案: 正本境界が二重化し運用上の誤参照を招くため不採用。
- DOMを全体構成直下に残す案: 「ARCHのみ直下」方針と矛盾するため不採用。
