## 決定事項
- デプロイ境界は `通常配備 / 初回配備 / 緊急配備` の3モードで定義し、モードごとに許可操作と再実行範囲を固定する。
- 配備責務は `BE / FE / Infra / DB / Doc / TestAsset` の6単位で管理する。
- IAMは `環境軸（dev/prod） x 責務軸（deploy/ops/audit）` のマトリクスで分離する。
- DDのエラー正本はAPI配下へ集約し、横断エラー文書は参照互換用途へ縮退する。
- DDのセキュリティ正本はINF配下へ統一し、SYS配下文書は廃止して参照互換のみ維持する。

## 理由
- 配備手順が領域記述と運用モード記述で分散しており、切戻し判断が文書横断になっていた。
- 権限分離を責務軸だけで管理すると、環境ごとの誤配備リスクを抑止しにくい。
- DDの正本分散がレビュー観点の重複を生み、変更漏れの原因になっていた。

## 影響
- `[[BD-INF-DEP-005]]` に3モード境界と6分類責務を追加する。
- `[[BD-INF-PLAT-001]]` に監視/イベント責務を追記し、INF全体像の欠落を解消する。
- `[[BD-SYS-QUAL-001]]` を App/Infra・BE/FE の責務マトリクスへ再編する。
- `[[DD-INF-SEC-003]]` を IAMマトリクス正本へ更新する。
- `[[DD-APP-ERR-001]]`, `[[DD-SYS-SEC-001]]` は廃止して参照互換文書へ更新する。

## 却下した選択肢
- 既存の4分類（docs/front/backend/infra）を維持する案: DB/TestAssetの配備責務が曖昧なまま残るため不採用。
- IAMを責務軸のみで管理する案: 環境境界の誤操作を抑止できないため不採用。
