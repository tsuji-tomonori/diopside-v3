## 設計方針
- 色設計は現行実装 `web/src/styles/global.css` のトークン値を基準に定義する。
- 配色判断は見た目優先ではなく、コントラスト閾値と非色依存の成立を優先する。

## 基準トークン（現行実装準拠）
| トークン | 用途 | 基準値 |
| --- | --- | --- |
| `--bg` | ページ背景 | `#f7f6f2` |
| `--panel` | パネル背景 | `rgba(255,255,255,.96)` |
| `--text` | 本文色 | `#111114` |
| `--muted` | 補助テキスト | `rgba(17,17,20,.68)` |
| `--border` | 境界線 | `rgba(17,17,20,.32)` |
| `--accent` | 主要操作色 | `#6E3FE7` |
| `--focus` | フォーカスリング | `0 0 0 3px rgba(110,63,231,.25)` |

## コントラスト規約
- 通常テキストは4.5:1以上を下限とする（[[RQ-UX-004-02]]）。
- UI境界と操作対象は3:1以上を下限とする（[[RQ-UX-004-02]]）。
- アウトラインボタンは背景とボタン内部背景の双方に対して3:1以上を下限とする（[[RQ-UX-025-04]]）。
- 前景色を指定する場合は背景色を必ずセット指定する（[[RQ-UX-025-06]]）。
- 状態伝達は色のみ禁止とし、アイコンまたは文言を併用する（[[RQ-UX-004-01]], [[RQ-UX-004-03]]）。
