## 設計方針
- 開発環境は、実装・検証の反復速度を優先しつつ、本番反映経路（Quartz + CDK）と同一の設計原則を維持する。
- システム実行基盤は [[RQ-PC-005]] に従い AWS を正本とし、ローカル実行は補助環境として扱う。

## 環境定義
- 実行主体: 開発者ローカル + AWS（開発用アカウント/開発用スタック）。
- 主要用途: 実装、単体確認、文書検証、配信手順の事前確認。
- データ方針: 本番相当構造のサンプルデータを使用し、本番データの直接編集を禁止する。

## 必須依存
- IaC/配信: AWS CDK、S3、CloudFront、`task docs:deploy` 実行チェーン。
- ドキュメント: Obsidian Markdown、Quartz build（`quartz/public` 出力）。
- 品質ゲート: `lint` / `test` / `build` / `cdk synth` を開発段階で再現可能とする。

## 運用上の制約
- Construct/Stack 内で `process.env` を直接参照せず、Stage/props 経由で環境差分を注入する（[[RQ-DEV-001]] 整合）。
- 秘密情報はリポジトリへ保存せず、認証情報はローカル安全ストアまたはAWS側で管理する（[[RQ-SEC-001]] 整合）。
- 検証用途は常設の独立環境を持たず、本番準拠の一時環境を都度構成して確認する。
- 一時検証の結果は [[IT-ENV-001]] と受入関連文書へ記録し、設計本文と運用記録を分離する。
- インフラ詳細は [[BD-INF-002]] と [[DD-IAC-001]] を正本とし、本書は環境要件の入口として扱う。
