## 目的
- diopsideの結合テストをUC基準で設計し、`RQ-FR-*` を主因子とするIT-PW文書を整備する。

## このスキルを使う条件
- `docs/5.結合テスト(IT)/12.ペアワイズ(PW)` 配下の `IT-PW-*` を新規作成または更新するとき。
- `IT-PLAN-001` のUC基準方針に従い、`RQ-UC-001` から `RQ-UC-009` の因子モデルを定義するとき。
- `IT-CASE-*` の因子根拠として、UCごとの組合せ条件を明示するとき。

## このスキルを使わない条件
- 個別ケースの手順や期待結果のみを更新する場合（`doc-it-case` を使う）。
- 結合テスト計画（体制/範囲/完了条件）のみを更新する場合（`doc-it-plan` を使う）。

## 出力契約
- 出力対象は `docs/5.結合テスト(IT)/12.ペアワイズ(PW)/IT-PW-*.md` とし、`filename == id` を維持する。
- `IT-PW-001` はガイド文書として、適用範囲・因子設計ルール・運用手順を記載する。
- `IT-PW-UC-*` は `pairwise` コードブロックで `meta` と `factors` を定義する。
  - `meta.id` / `meta.uc` / `meta.cases`
  - `factors`（因子名は日本語、機能要件ベース）
  - 必要時に `excludes`

## 記述ルール（IT-PW）
- 対象UCは `RQ-UC-001` から `RQ-UC-009` とし、`RQ-UC-010` から `RQ-UC-013` はIT-PW対象外とする。
- 因子は `RQ-FR-*` を主因子、APIパラメータ・外部依存・データ状態を補助因子として定義する。
- 標準強度は2-wise、冪等性/同時実行制約/障害復旧導線は3-way補完を許容する。
- 異常系は1ケース1異常要因を原則とし、エラー優先順位のマスキングを避ける。

## 品質チェック
- `IT-PW-UC-*` の `meta.cases` が対応 `IT-CASE-*` と一致することを確認する。
- `task docs:guard` を実行し、リンク整合とfrontmatter整合を確認する。
- `task docs:trace` を実行し、RTM静的ビューを更新する。
